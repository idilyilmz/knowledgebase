## 1. Introduction

### 1.1 Why this topic matters

More and more businesses are using LLMs as solutions to their problems. Less people and more machines. There's nothing wrong with that, but we need to be better at understand how these LLMs work. And *how we can build an application that uses a Large Language Model API, and what design choices shape whether it works well?*. So we should do more than just use Models like (ChatGPT, Claude and Gemini). Everyone works with these agents, but how do they actually work? How are they actually build? That's a core skill for AI Engineers these days.

### 1.3 Central & Sub-Questions

This paper discusses the following question, *How do you build an application that uses a Large Language Model API, and what design choices shape whether it works well?*. It will be methodically explored by also answering the following subquestions:
- Whats a LLM API?
- How do you make your first API call from Python?
- What is prompt engineering?
- How do you handle multi-turn conversations?
- How do you evaluate whether an LLM is giving you good outcomes?
- What’s the next layer up?
