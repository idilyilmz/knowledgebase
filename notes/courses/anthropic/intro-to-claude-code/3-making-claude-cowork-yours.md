## Plugins: Cowork as a specialist

This lesson introduces **plugins** — bundles of domain expertise that let Cowork approach tasks the way a specialist in your function would.

**What's in a plugin:**
- **Skills** — Instructions for specific workflows (e.g., how to structure a deal brief). Claude uses them automatically or you can invoke them with `/` in the prompt.
- **Connectors** — Links to the systems where the work happens (CRM, docs, messaging).
- **Subagents** — Parallel workers for specialized tasks (e.g., one per account in a book-wide review).

Open-source plugins exist for most knowledge-work roles: sales, marketing, product, finance, legal, ops, support, data, and more. They work as-is but can be modified.

**Installing one** is three steps: browse in Cowork's Customize area, click install, and optionally customize. Once installed, the plugin is just a folder on your machine with plain-text files — no build step, so editing a skill means opening its file and changing it.

**About skills:** They're the core building block inside plugins — markdown files that teach Claude how to handle one thing (a workflow, format, or process). Skills aren't Cowork-specific; they work across all Claude surfaces. A plugin is simply the Cowork-specific way of bundling skills with the connectors they need.

**Your practice task:** browse the Customize area, install the plugin closest to your role, and open one of its skill files to see that it reads like a brief you'd give a teammate.
